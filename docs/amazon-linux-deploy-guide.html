MT5 Risk Monitor PRO

Amazon Linux 生产环境部署流程

目标：用同一个生产 zip 包完成首次部署和后续系统更新。Node.js 不打包进应用，由安装脚本在线安装 Node.js 20 LTS。

推荐方案

最推荐：GitHub Release 上传生产 zip 包，Amazon Linux 服务器通过一键脚本下载并安装。

原因：服务器不需要前端构建环境，不需要拉取完整源码，不会上传 config.local.json、.env、登录用户、MT5 密码等本地数据。

方式适合场景建议
GitHub Release zip生产部署、升级、回滚推荐
Git clone 源码开发机或 CI 构建可以，但服务器需要更多工具和构建步骤
手动 SCP zip无 GitHub 或内网机器可用，执行包内安装脚本即可

一、本地生成生产部署包

在 Mac 本地项目目录运行：

cd "/Users/leo/Documents/爆仓提醒"
chmod +x scripts/make_production_package.sh scripts/install_amazon_linux.sh
scripts/make_production_package.sh

生成结果：

dist/mt5-risk-monitor-production.zip
dist/mt5-risk-monitor-<version>-<timestamp>.zip

mt5-risk-monitor-production.zip 是固定文件名，适合 GitHub Release 的 latest download URL。

二、上传到 GitHub

方式 A：GitHub 网页上传 Release

  1. 创建一个 Private GitHub 仓库，例如 OWNER/mt5-risk-monitor。
  2. 把源码推到仓库，至少需要包含 scripts/install_amazon_linux.sh。
  3. 进入 GitHub 仓库的 Releases，创建新版本，例如 v0.1.0。
  4. 上传 dist/mt5-risk-monitor-production.zip 到 Release assets。

方式 B：命令行上传 Release

本地安装 GitHub CLI 后：

gh auth login
gh repo create OWNER/mt5-risk-monitor --private --source . --remote origin --push
gh release create v0.1.0 dist/mt5-risk-monitor-production.zip \
  --repo OWNER/mt5-risk-monitor \
  --title "MT5 Risk Monitor v0.1.0" \
  --notes "Production deployment package"

三、Amazon Linux 一键安装

登录 EC2：

ssh -i your-key.pem ec2-user@你的服务器IP

公开仓库或公开 Release

curl -fsSL https://raw.githubusercontent.com/OWNER/mt5-risk-monitor/main/scripts/install_amazon_linux.sh \
  | sudo bash -s -- --github-repo OWNER/mt5-risk-monitor

私有仓库或私有 Release

创建 GitHub fine-grained token，只给这个仓库的 Contents/Metadata 读取权限，然后：

export GITHUB_TOKEN="ghp_xxxxxxxxxxxxxxxxxxxx"
curl -fsSL -H "Authorization: Bearer $GITHUB_TOKEN" \
  https://raw.githubusercontent.com/OWNER/mt5-risk-monitor/main/scripts/install_amazon_linux.sh \
  -o /tmp/install_mt5_risk_monitor.sh
sudo -E bash /tmp/install_mt5_risk_monitor.sh --github-repo OWNER/mt5-risk-monitor

如果你已经拿到了 zip 的直接下载地址，也可以：

sudo bash /tmp/install_mt5_risk_monitor.sh \
  --package-url "https://github.com/OWNER/mt5-risk-monitor/releases/latest/download/mt5-risk-monitor-production.zip"

四、安装脚本会自动完成什么

  • 在线安装 Node.js 20 LTS。
  • 安装 curl、unzip、tar 等基础工具。
  • 创建 Linux 服务用户 mt5risk。
  • 部署应用到 /opt/mt5-risk-monitor。
  • 首次部署时复制默认配置为 config.local.json。
  • 首次部署时生成随机管理员账号和强密码，密码只显示一次。
  • 创建并启动 systemd 服务 mt5-risk-monitor。
  • 如果填写域名，安装 Nginx 并反向代理到本地应用端口。

五、部署过程中的交互项

脚本会按“下一步”方式询问：

项目默认值说明
安装目录/opt/mt5-risk-monitor保存应用和配置。
服务名mt5-risk-monitorsystemd 服务名。
服务用户mt5risk低权限运行用户。
端口2001直接用 IP 访问时打开这个端口。
域名空填写后自动配置 Nginx 80 端口反代。

六、访问方式

IP + 端口

http://服务器IP:2001

需要在 EC2 Security Group 放行 TCP 2001，只建议放行你的固定 IP。

域名 / Cloudflare

安装时填写域名后，访问：

http://你的域名

Cloudflare 常规代理不支持任意 2001 端口，因此域名模式建议使用 Nginx 监听 80/443，再反代到本地 2001。Cloudflare DNS 记录指向 EC2 公网 IP 并开启 Proxied。

七、常用运维命令

sudo systemctl status mt5-risk-monitor
sudo systemctl restart mt5-risk-monitor
sudo journalctl -u mt5-risk-monitor -f
sudo cat /opt/mt5-risk-monitor/config.local.json

八、升级流程

  1. 本地运行 scripts/make_production_package.sh 生成新的 zip。
  2. 上传 zip 到 GitHub Release，覆盖或新建 Release asset。
  3. 服务器再次执行同一个安装命令。

升级不会覆盖 config.local.json、.env、.env.local、.state 和已有登录用户。

九、安全注意事项

  • GitHub 仓库建议设为 Private。
  • 不要提交 config.local.json、.env、MT5 密码、Telegram Token。
  • 首次生成的管理员密码只显示一次，立即保存到密码管理器。
  • 公网访问建议使用 Cloudflare、Nginx HTTPS 或仅开放可信 IP。
  • EC2 Security Group 不要对全网开放 MT5 服务器端口，只需要本系统能访问 MT5 入口。
