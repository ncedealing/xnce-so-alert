MT5 Risk Monitor 远程登录与部署说明

当前可运行版本不依赖 Python：Node.js 服务 + React 静态前端 + 本地 JSON 配置 + scrypt 密码哈希 + HttpOnly Session Cookie

1. 当前版本技术栈

模块技术说明
后端Node.js 原生 HTTP不引入数据库，方便 Windows/Linux 部署。
前端React + Vite 静态构建构建后由同一个 Node 服务托管。
登录scrypt 密码哈希 + HttpOnly Cookie密码不明文保存，浏览器通过会话 cookie 登录。
配置config.local.json首次部署脚本自动写入 admin 用户。

2. 首次部署流程

检查环境

cd /path/to/爆仓提醒
npm run deploy:check

生成远程登录账号

npm run setup:remote-login

脚本会生成符合安全规范的随机用户名和强密码，并写入 config.local.json。密码只会在终端显示一次，请立即保存到密码管理器。

构建前端并启动服务

npm run ui:build
npm run web

远程访问

http://服务器IP:端口

默认端口来自 config.local.json 的 web.port。远程部署时把 web.host 改为 0.0.0.0，即可用 服务器IP:端口 访问。

如需域名访问，可使用 Nginx/Caddy/Cloudflare Tunnel 反向代理到本服务端口；Cloudflare HTTPS 请求会携带 X-Forwarded-Proto: https，系统会自动为登录 Cookie 增加 Secure 属性。

3. 后台账号管理

登录后进入 Configuration -> Security：

  • 修改当前 admin 密码。
  • 添加新的 admin 用户。
  • 用户名要求：以字母开头，长度 8-32，只能包含字母、数字、下划线和中划线。
  • 密码要求：至少 12 位，并包含大写字母、小写字母、数字和符号。

4. 生产注意事项

建议使用 Nginx/Caddy 反向代理并启用 HTTPS。不要把未加密 HTTP 直接暴露在公网。正式生产技术栈建议见 docs/production-stack-recommendation.md。
  • 限制服务器防火墙，只开放 Web 端口给可信 IP。
  • 使用 Cloudflare 代理时，建议源站防火墙只允许 Cloudflare IP 或内网管理 IP。
  • MT5 Manager 密码建议保存在环境变量或后台密码字段，保存后 UI 会隐藏。
  • Telegram Bot Token 和 SMTP 密码同样不要写入公开仓库。
  • 升级后先运行 npm run deploy:check，确认测试和前端构建通过。
  • 如果浏览器提示接口不存在，通常是旧服务未停止，重新运行启动脚本或停止旧进程。

5. 常用命令

# Mac 本地测试并自动打开浏览器（Python 只用于本地 demo）
python3 mac_local_demo.py --stop-stale

# 生产初始化远程登录
npm run setup:remote-login

# 重置并重新生成 admin
npm run setup:remote-login -- --reset

# 指定用户名
npm run setup:remote-login -- --username admin_ops01

# 部署检查
npm run deploy:check
